## git tag

El comando `git tag` se usa para **marcar versiones importantes** en el historial, como “v1.0” o “release-final”.  
Esto ayuda a identificar commits clave y organizar entregas.

## Crear un tag
git tag v1.0

## Subir el tag al remoto

git push origin v1.0


## Listar tags existentes

git tag.
